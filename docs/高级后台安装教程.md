# ZTE U60 Pro 高级面板安装教程

## 简介

一个独立的 Web 高级管理面板，运行在路由器 8888 端口，提供比原生 WebUI 更丰富的网络管理功能。

**基于 [ZTE-Script-NG](https://github.com/tpoechtrager/ZTE-Web-Script) 的 ubus API 提取重构，感谢原作者 Thomas Pöchtrager。**

### 功能列表

| 功能 | 说明 |
|------|------|
| 📡 网络信息 | 运营商、连接类型、频段、带宽、Cell ID |
| 📶 信号详情 | 每个 NR/LTE CA 载波的 RSRP/RSRQ/SINR/RSSI/PCI/BW/频点 |
| 📊 流量统计 | 实时速度、当月流量、总流量 |
| 🌐 WAN 状态 | IPv4/IPv6 地址、网关、DNS（双栈完整信息） |
| 💻 系统状态 | CPU 温度/负载、内存、运行时间 |
| 🌡️ 温控开关 | 设备过热自动降速控制（开启/关闭） |
| 📻 网络制式 | 5G SA / 5G NSA / 4G+3G / 仅4G / 仅3G 切换 |
| 🔒 4G 频段锁 | B1-B71 共 24 个频段，支持单频/组合/手动输入 |
| 🔒 5G 频段锁 | N1-N79 共 21 个频段 |
| 📍 小区锁定 | 5G/4G 小区锁定/解锁（PCI + ARFCN） |
| 📶 WiFi 设置 | 发射功率、国家码、最大连接数（内联输入框） |
| 📋 详细信息 | WiFi 参数、硬件/软件、SIM 卡、WMS 全部内联展示 |

### 特性

- **单文件 HTML**（~51KB），无外部依赖
- **响应式设计**，手机和电脑均可使用
- **密码缓存**，首次输入后自动登录
- **Session 自动续期**，过期后静默重新登录
- **持久化安装**，重启自动启动
- **纯 JS SHA256**，不依赖 HTTPS

## 前提条件

1. **已获取 SSH 访问权限**（通过 ADB 或其他方式部署了 dropbear）
2. 路由器上存在 `/overlay/home/scoltc` 目录（SSH 用户的 home）
3. ADB 连接可用（用于首次安装）

## 安装方法

### 方式一：一键脚本（推荐）

```bash
# 1. 将文件推送到路由器
adb push index.html /tmp/index.html
adb push install.sh /tmp/install.sh

# 2. 执行安装脚本
adb shell 'sh /tmp/install.sh'
```

### 方式二：手动安装

```bash
# 1. SSH 登录路由器
ssh u60pro

# 2. 创建目录
mkdir -p /overlay/home/scoltc/web-panel

# 3. 退出 SSH，推送文件
adb push index.html /overlay/home/scoltc/web-panel/index.html

# 4. ADB shell（需要 root 做以下操作）
adb shell

# 5. 创建手机兼容文件
cp /overlay/home/scoltc/web-panel/index.html /overlay/home/scoltc/web-panel/mobile.html
cp /overlay/home/scoltc/web-panel/index.html /overlay/home/scoltc/web-panel/moible.html

# 6. 放行防火墙
iptables -I INPUT 1 -p tcp --dport 8888 -j ACCEPT

# 7. 复制到 /tmp 并启动
mkdir -p /tmp/zte-panel
cp /overlay/home/scoltc/web-panel/*.html /tmp/zte-panel/

start-stop-daemon -S -b -m -p /tmp/zte-panel.pid \
    -c 1001:1001 \
    -x /usr/sbin/uhttpd -- \
    -p 0.0.0.0:8888 \
    -h /tmp/zte-panel \
    -x /cgi-bin \
    -u /ubus \
    -U /var/run/ubus/ubus.sock \
    -X -f -D -R \
    -n 3 -N 100 \
    -t 3600 -T 30 \
    -k 20 -A 1

# 8. 配置开机自启（编辑 /overlay/dropbear/start_ssh.sh，在 exit 0 前加入）
# sh /overlay/home/scoltc/web-panel/start_panel.sh &
```

## 访问

安装完成后，浏览器打开：

```
http://192.168.0.1:8888
```

输入路由器管理密码登录即可。

> ⚠️ **不要用 6666 端口**，该端口被 Chrome/Firefox/Safari 列为"不安全端口"，浏览器会拒绝连接。

## 技术细节

### 架构

```
浏览器 → http://192.168.0.1:8888 → uhttpd (radio用户)
                                        ↓
                                    /ubus/ 端点
                                        ↓
                                   ubus JSON-RPC
                                        ↓
                                  路由器内部服务
```

### 文件结构

```
/overlay/home/scoltc/web-panel/     # 持久化（重启不丢）
├── index.html                      # 主页面
├── mobile.html                     # 手机UA兼容（同内容）
├── moible.html                     # ZTE拼写错误兼容（同内容）
└── start_panel.sh                  # 启动脚本

/tmp/zte-panel/                     # 运行时（从overlay复制）
├── index.html
├── mobile.html
└── moible.html
```

### 为什么需要 mobile.html 和 moible.html？

ZTE 定制的 uhttpd 会检测浏览器 User-Agent，如果是手机访问，自动将请求路径重写为 `/mobile.html` 或 `/moible.html`（ZTE 的拼写错误）。必须提供这两个文件，否则手机访问会 404。

| 功能 | ubus service | method |
|------|-------------|--------|
| 登录获取 salt | `zwrt_web` | `web_login_info` |

| CPU 温度 | `zwrt_bsp.thermal` | `get_cpu_temp` |
| 温控开关 | `zwrt_bsp.thermal` | `get_policy` / `set_policy` |
| 设备信息 | `zwrt_mc.device.manager` | `get_device_info` |
| WAN 状态 | `zwrt_router.api` | `router_get_status` |
| 流量统计 | `zwrt_data` | `get_wwandst` |
| 制式切换 | `zte_nwinfo_api` | `nwinfo_set_netselect` |
| 4G 锁频 | `zte_nwinfo_api` | `nwinfo_set_gwl_bandlock` |
| 5G 锁频 | `zte_nwinfo_api` | `nwinfo_set_nrbandlock` |
| 4G 锁小区 | `zte_nwinfo_api` | `nwinfo_lock_lte_cell` |
| 5G 锁小区 | `zte_nwinfo_api` | `nwinfo_lock_nr_cell` |
| WiFi 配置 | `uci` / `zwrt_wlan` | `get` / `set` |
| 硬件信息 | `uci` | `get` (zwrt_common_info) |

### 网络制式对应关系

| 面板显示 | API 值 | 说明 |
|---------|--------|------|
| 自动 | `WL_AND_5G` | 设备自动选择 |
| 5G SA | `Only_5G` | 独立组网 |
| 5G NSA | `LTE_AND_5G` | 非独立组网（LTE 锚点+NR） |
| 4G/3G | `WCDMA_AND_LTE` | LTE + WCDMA |
| 仅 4G | `Only_LTE` | 仅 LTE |
| 仅 3G | `Only_WCDMA` | 仅 WCDMA |

## 卸载

```bash
# SSH 或 ADB shell
rm -rf /overlay/home/scoltc/web-panel
rm -rf /tmp/zte-panel

# 编辑 /overlay/dropbear/start_ssh.sh，删除以下两行：
# # 启动 ZTE 高级面板

# 杀掉运行中的进程
killall uhttpd  # 注意不要杀 80 端口的
```

## 常见问题

**Q: 浏览器显示 ERR_UNSAFE_PORT**
A: 你可能在用 6666 端口。改用 8888。

**Q: 手机显示 "Not Found /moible.html"**
A: 确保 `moible.html` 和 `mobile.html` 文件都存在于 web 目录中。

**Q: 操作后提示 "获取 salt 失败"**
A: 制式切换等操作会导致网络短暂断开，session 过期。面板会自动重新登录。如果仍然失败，手动刷新页面。

**Q: 重启路由器后面板不见了**
A: 检查 `/overlay/dropbear/start_ssh.sh` 中是否有 `start_panel.sh` 的自启动行。

**Q: WiFi 设置/频段锁定不生效**
A: ZTE 的 ubus 服务偶尔会返回 Access Denied（已知 bug），面板会自动重试最多 5 次。
